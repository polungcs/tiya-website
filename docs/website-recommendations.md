# 移青陣官方網站 改善建議報告

> 撰寫日期：2026-05-17
> 用途：給協會例會討論、後續網站改版規劃的依據
> 目前線上版本：<https://tiya-website.leepolung1.workers.dev/>

---

## 執行摘要（給時間有限的讀者）

1. **目前網站已上線**，有 8 個主要區塊，繁中／英文雙語，每次有 push 改動就會自動部署（push＝把改好的程式碼上傳到 GitHub，後台會自動把新版站台搬上線）。
2. **擁有自己的官網對協會的核心價值**：訊息主導權、長期內容資產、媒體與政府單位的對接正門、不被社群平台演算法綁架。
3. **短期可以加強**：影響力數據區塊、成員故事頁、電子報訂閱、「加入我們」依身份分流（個人／教育者／媒體／企業）。
4. **中期等取得勸募字號後**：完整捐款流程、月定額會員、財務透明頁。
5. **長期願景**：東南亞語系擴充、互動式年度報告、Podcast／影音故事典藏（呼應「留聲機」品牌定位）。

## 一、現況盤點 — 我們已經做了什麼

### 1.1 網站架構

| 區塊 | 內容 | 狀態 |
|---|---|---|
| Hero 首屏 | 品牌標語、組織英文名、簡介一段 | ✅ |
| About 關於我們 | 組成介紹、來源國家、核心價值（對話／人權／倡議／共榮） | ✅ |
| News 媒體報導 | 卡片式、左右滑動，目前 2 篇（公視、想想論壇） | ✅ |
| Events 實體活動 | 卡片式，Facebook 貼文嵌入，目前 2 場 | ✅ |
| Column 天下獨立評論專欄 | 3 篇文章＋縮圖＋「我要投稿」CTA（Call to Action，呼籲動作） | ✅ |
| Donate 捐款資訊 | 法律免責聲明、匯款帳戶（戶名／銀行／帳號目前都標「待確認」） | ⚠ 待協會帳戶開戶後填入 |
| Social 社群連結 | Facebook、Instagram、Email | ✅ |
| Footer 頁尾 | 聯絡、社群、版權 | ✅ |

### 1.2 技術基底

- **Astro**：靜態網站框架，build（編譯）時就把所有頁面變成純 HTML，速度快、不用後端伺服器、不用資料庫。
- **內容管理**：活動、專欄、新聞都是 markdown（一種純文字格式，類似簡化版的 Word）放在 `src/content/` 資料夾，新增一篇就是新增一個檔案。
- **部署**：GitHub repo → push 到 main 分支 → Cloudflare Workers 自動 build 上線，1-2 分鐘內生效。零月費。
- **多語**：目前繁中、英文兩個版本，共用同一套元件。

### 1.3 法律邊界（重要前提）

協會**尚未取得勸募字號**，所以網站上：
- ✅ 可以做：被動提供匯款資訊、收據說明、匯款後通知表單。
- ❌ 不能做：「Donate Now」「支持我們」這類號召性大按鈕、捐款金額示意（「100 元＝場地費」這類）、任何「請捐款」訊息。

這條約束會影響本報告中第四部分關於募款功能的時程規劃。

## 二、為什麼擁有屬於自己的官網很重要

社群帳號（FB、IG）已經能把訊息發出去，為什麼還需要花力氣維護一個官網？以下是**官網能做、社群做不到**的事：

### 2.1 訊息主導權

社群平台的演算法不是協會控制的。FB 一次調整，觸及率可以掉到原本的 10%。官網的版面、字句、出現順序、保留多久——全由協會自己決定。**Meta 哪天封了我們的帳號，IG 哪天倒了，故事還在自己的網域上。**

### 2.2 對外的「正門」

當記者要寫一篇關於新二代的報導、當立委助理要找移民議題的引述、當學者要做學術引用、當企業要找 CSR 合作對象——他們第一件事是 Google「台灣移民青年倡議陣線」。
**他們點進去看到的是 FB 還是官網，給的信任感差很多。** 官網 = 協會「正式存在」的招牌。

### 2.3 SEO（搜尋引擎最佳化）

社群貼文在 Google 搜不太到（FB 不開放給 Google）。官網上的每一篇文章、每一次活動，都是長期被搜尋到的入口。
舉例：未來有民眾搜「新二代 身分認同」「移民工 勸募」「東南亞 第二代 台灣」——官網有相關內容就會出現。
**內容會累積成長期資產，每篇貼文是抵 IG 限時動態 24 小時。**

### 2.4 國際對話的入口

英文版讓我們有資格出現在國際移民倡議、人權報告、跨國 NGO 合作場合。
這在 IG／FB 上很難做到——對國際讀者來說，他們不熟悉中文社群平台，但他們會看英文網頁。

### 2.5 內容是資產，不是即時動態

社群貼文的生命週期是「幾天」。網站上的成員故事、年度報告、活動紀錄是「永遠」。
五年後新進的工作者、新的合作夥伴、新的研究者，都還能完整看到協會的歷程。

### 2.6 招待身份分流的工具

不同人來找我們的目的不同：個人想了解、媒體要採訪、學者要訪談、企業要合作、移民家庭想求助。
社群只能一條時間軸對所有人。官網可以做出**「我是 ___，請點這裡」**的入口分流（rethinktw 就是這樣，他們的選單裡有「我是個人／我是教育者／我是企業」）。

## 三、改進建議 — 針對現有區塊

### 3.1 Hero 首屏

**目前狀況**：簡介段落以「我們是誰」描述為主。
**建議**：加一句**情感入口**式的標語，像 rethinktw 的「讓環保行動成為你我日常」。
TIYA 可考慮類似方向：
- 「讓每一個新二代的聲音都被聽見」
- 「我們不是『新』台灣人，我們就是台灣人」
- 「家鄉是兩個地方，認同是一個自己」

要協會內部討論定調。

### 3.2 About 關於我們

**目前狀況**：已經有來源國家、核心價值。
**建議**：加一個小區塊「我們目前有 N 位成員，來自 9 個原生國家」——具體數字會比抽象描述有力。

### 3.3 News 媒體報導

**目前狀況**：2 篇卡片，左右滑動，圖片為主。
**建議**：
- 每多一篇報導就累積一張卡，**目標年內累積 10+ 篇**，視覺上更有「常被報導」的份量。
- 考慮加一個「下載媒體採訪聯絡單」按鈕——讓記者直接拿到聯絡窗口、組織簡介、可受訪議題清單、新聞稿模板。

### 3.4 Events 實體活動

**目前狀況**：嵌入 FB 貼文。
**建議**：每場活動建一個獨立頁面（不只嵌入 FB），包含：
- 完整描述
- 講者介紹
- 現場照片 5-10 張
- 觀眾回饋摘要
- 影音（如果有錄）
這樣將來 Google 搜「TIYA 宜蘭場」「移民青年 對話活動」搜得到內容。

### 3.5 Column 天下獨立評論專欄

**目前狀況**：3 篇文章＋縮圖＋投稿 CTA。
**建議**：
- 文章累積到 10+ 篇後考慮加「依議題分類」（升學、家庭、認同、勸募、政策…）。
- 投稿區塊可以加一句「我們提供寫作指導，新手也可以投」——降低門檻，鼓勵成員投稿。

### 3.6 Donate 捐款資訊

**目前狀況**：法律免責 + 匯款資訊（待填）。
**建議（取得勸募字號前）**：
- 帳戶開戶完成後立即把「待確認」換成實際資訊。
- 加「匯款後通知表單」（Google Form 嵌入），方便對帳。
- 加 PDF 收據說明。

**建議（取得勸募字號後）**：詳見第四部分 4.2。

### 3.7 Social / Footer

**目前狀況**：FB、IG、Email。
**建議**：考慮加入 **LINE 官方帳號**（台灣使用者主要管道）、**Threads**（年輕族群正在遷移）、未來 **YouTube** 或 **Podcast** 頻道。

## 四、新功能建議 — 可以新增什麼

依**時程**分三組。

### 4.1 短期（1-2 個月內、不需協會額外決策可以動）

#### A. 影響力數據區塊
參考 rethinktw 的「我們的影響力」。內容例如：
- 已舉辦 N 場實體對話
- 累積 N 位成員
- 被 N 家媒體報導
- 天下專欄總閱讀數 N 萬次
- 來源國家：9 個

**為什麼重要**：對贊助者、媒體、合作對象來說，**具體數字比抽象描述更有力**。也讓協會成員自己看到累積的成果，士氣激勵。

#### B. 成員故事頁（「故事牆」）
每位成員一張卡片：照片、原生國家、一段自述、一句最想對台灣社會說的話。
這是 TIYA 的**核心差異化**——「新二代留聲機」名字本身就是「記錄聲音」的意思。網站上應該有最直接的體現。

#### C. 電子報訂閱
表單放在 Footer 或單獨區塊。每月或每季發一封：本月活動、新文章、媒體報導、會員招募訊息。
**工具建議**：[Buttondown](https://buttondown.com/) 或 [Substack](https://substack.com/) 免費起步，比自建省事。

#### D. 「加入我們」依身份分流
建一個 `/join` 頁面，三個入口：
- 我是新二代（想加入會員 / 投稿故事 / 來聊聊）
- 我是研究者／媒體（想採訪 / 學術合作）
- 我是教育者／NGO 同伴（想合辦活動）

每個入口導到對應的 Google Form 或 mailto。

#### E. FAQ 常見問題頁
回答這些：「新二代的定義？」「TIYA 是什麼樣的組織？」「如何加入？」「如何投稿？」「為什麼還不能線上捐款？」
為 SEO 加分，也減輕內部回覆訊息的工作量。

### 4.2 中期（取得勸募字號後 / 1-2 季）

#### A. 完整捐款流程
- 線上金流（信用卡 / Apple Pay / Line Pay）。建議走 **NetiCRM**（rethinktw 也用這個，NGO 友善的開源捐款系統）或 **奉獻箱 OurStable** 等 NGO 平台。
- 捐款層級示意（「500 元支持一場對話的茶水費」、「3000 元贊助一位新二代來回機票」等）。
- 月定額會員方案。
- 收據自動寄送。

#### B. 透明財務報告頁
每年年底或年中發布。包括：
- 收入來源（補助金、個人捐款、企業合作各佔多少）
- 支出明細（人事 / 活動 / 行政 / 倡議）
- 一張視覺化的圓餅圖
**透明 = 公信力**。Charity: Water 是這個模式的國際典範。

#### C. 年度成果報告（網頁版）
不要做成 PDF 然後叫人下載——做成**互動式網頁**，含：
- 數據圖表
- 該年度精選故事
- 重要對外發聲時刻
- 來年方向預告

### 4.3 長期願景（半年以上）

#### A. 東南亞語系擴充
協會服務的群體本來就涵蓋越南、印尼、泰國、菲律賓等新住民家庭。網站若能有越南語、印尼語等版本，至少首頁＋關於＋聯絡，象徵意義非常大。
**門檻**：翻譯需要找母語者協助，目前框架（Astro 多語）技術上可以支援。

#### B. Podcast / 影音故事典藏
協會名稱叫**「留聲機」**——這本來就是聲音的隱喻。長期可以做：
- 每月一集成員口述故事 Podcast
- 整理對話活動的現場錄音精選
- 嵌入 Spotify / Apple Podcasts
**國際參考**：[StoryCorps](https://storycorps.org/)（美國民間口述歷史 NGO，純粹靠錄音故事撐起一個組織）。

#### C. 與議題搜尋連動的內容地圖
網站上的所有內容（文章、活動、故事）依議題（升學、勸募、認同、家庭、政策）交叉索引。
讀者點「政策」可以看到所有相關文章＋活動＋成員故事。

## 五、參考案例

### 5.1 國內

#### 重新思考環境教育協會 RE-THINK（<https://rethinktw.org/>）
**值得學習**：
- 三層導覽：開始和行動 / 環保行動 / 文章。把訪客依「我想知道現況」「我想參與」「我想看文章」分流。
- 「加入我們」分三個身份入口（個人／教育者／企業）。
- 「我們的影響力」用具體數字（清理量、社群觸及、課程觸及）。
- FAQ 分四類（淨灘 / 捐款 / 課程申請 / 關於組織）。
- 部落格分五類（環保新知 / 大對決 / 人物專訪 / 年度成果 / 企業合作）。

**對 TIYA 的啟發**：身份分流、影響力數據、議題化的內容組織。

#### 婦女救援基金會（<https://www.twrf.org.tw/>）
**值得學習**：
- 議題（人口販運、性暴力、慰安婦、新移民）每個都有獨立的議題頁＋資源庫。
- 故事為核心：當事人匿名敘事、影像紀錄。
- 線上捐款做得完整（取得勸募字號的協會可參考）。

### 5.2 國際

#### Define American（<https://defineamerican.com/>）
**為什麼參考**：美國移民倡議 NGO，由移民出身的記者 Jose Antonio Vargas 創辦。**和 TIYA 任務最像**。
**值得學習**：
- 用個人故事推動議題的方式。
- 與媒體（電視劇、新聞媒體）的合作模式：他們有專屬的「Hollywood 合作」頁面，幫助影視作品準確呈現移民議題。

#### StoryCorps（<https://storycorps.org/>）
**為什麼參考**：純口述歷史 NGO，名字也跟「留聲機」精神共鳴。
**值得學習**：每段錄音都有對應的網頁卡片＋文字摘要＋背景介紹。是「聲音故事典藏」的標竿。

#### Charity: Water（<https://www.charitywater.org/>）
**為什麼參考**：透明財務的國際典範。
**值得學習**：100% 模式（所有捐款直接用於專案，組織營運由另一群長期贊助者支付）。雖然 TIYA 規模不同，但「**捐款人能清楚看到錢去哪**」的精神可以借鏡。

## 七、給例會的討論題

**需要協會決策的：**
1. Hero 標語要不要定一個正式的對外標語？目前是描述句，缺一句口號式的。
2. 成員故事頁要不要做？如果做，採用哪些成員、用什麼方式呈現？是否每位成員都有意願公開照片＋姓名？
3. 「加入我們」表單由誰負責回覆？依身份分流到不同窗口的話，每個窗口要指派誰？
4. 電子報發送頻率（月 / 季）？由誰寫？
5. 取得勸募字號的時程預期？這會影響網站捐款區的改版時程。

**已經可以動、不需例會決議的：**
1. 影響力數據區塊——只要協會把數字整理出來就可以做。
2. FAQ 雛形——我先整理一份草稿，再請協會校稿補充。
3. News／Events 區塊持續累積——每多一篇就更新。

## 附錄：例會呈現形式建議

你問到例會要用簡報還是文件——兩個用途不同：

| 形式 | 適合的場合 | 對 TIYA 例會 |
|---|---|---|
| **投影片** | 需要簡報主講、控制節奏、視覺重點突出 | 適合：在會議現場由你主講「為什麼要改、改哪些、哪些要協會決定」 |
| **完整文件**（這份） | 需要會後留檔、深度閱讀、跨會次參考 | 適合：會前傳給大家自行閱讀；會後協會行政留底 |
| **兩者並用** | 文件當前置閱讀＋會議用精簡簡報＋會後 follow up | **建議這個方式** |

如果採「兩者並用」，動作會是：
