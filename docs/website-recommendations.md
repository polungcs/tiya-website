## 執行摘要（給時間有限的讀者）

1. **網站目前可正常瀏覽，但還沒正式上線**。已有 8 個主要區塊，繁中／英文雙語，每次有改動就會自動部署（push＝把改好的程式碼上傳到 GitHub，後台會自動把新版站台搬上線）。
2. **協會帳戶已開戶**，捐款區只差填入實際帳戶資訊就可以對外。
3. **擁有自己的官網對協會的核心價值**：訊息主導權、長期內容資產、媒體與政府單位的對接正門、不被社群平台演算法綁架。
4. **短期可加強**（不需大筆資源）：成員故事頁、FAQ、後台流量觀察。
5. **中期等取得勸募字號後**：完整捐款流程、年度成果報告網頁版。
6. **長期願景**：東南亞語系擴充、Podcast／影音故事典藏 — 都是「有資源時可以做」的方向，非必做。

### 1.2 技術基底（含一些對非工程背景的解釋）

- **Astro**：靜態網站框架。
  - 「靜態 vs 動態」白話解釋：**靜態網站**就像一本印好的書，每個人看到的內容都一樣，網站打開很快、不用付主機月費、不會被駭。**動態網站**像每次客人來都重新做一份料理，需要後端伺服器和資料庫（多人帳號、即時留言、購物車這類功能會需要動態）。我們網站目前的需求（介紹、文章、捐款資訊）都用靜態就夠了，所以選 Astro。
  - 一旦未來真的需要會員登入、線上募款金流回呼之類的功能，**也可以混搭**：把絕大多數頁面保持靜態、少數需要互動的頁面接後端服務。
- **內容管理**：活動、專欄、新聞都是 markdown（一種純文字格式，類似簡化版的 Word）放在 `src/content/` 資料夾，新增一篇就是新增一個檔案。
- **部署管線**：GitHub repo → push 到 main 分支 → Cloudflare Workers 自動 build 上線，1-2 分鐘內生效。零月費。
- **多語**：目前繁中、英文兩個版本，共用同一套元件。

**🔴 重要待討論：未來租用正式網域**

換掉現在的 `tiya-website.leepolung1.workers.dev`、改成像 `tiya.org.tw` 這樣的正式網域，是讓官網「正式存在」的最後一步。這節整理協會需要做的決定，給例會用。

> **2026-05-19 例會後備註**：協會目前沒預算可立刻 cover 網域費，網站上架時程也沒有急迫性。本節先看「過渡方案」即可，A-I 為「未來有預算或拿到補助時」的完整討論。

#### 過渡方案 ｜ 沒預算也能上線（0 元，現況可立刻優化）

**現況**：網站已經能用 — 跑在 `tiya-website.leepolung1.workers.dev` 上，完全免費。這跟政大某個移民議題課程網站（`nccu-immigrants-digital-marketing.weebly.com`）的做法是**同一個概念**：用平台給的免費子網域。

不一定要立刻買網域。短期內走過渡方案，補助下來或預算到位再升級。

**為什麼 Cloudflare 願意這麼大方？有什麼代價？**

Cloudflare 是上市公司，免費版的存在**不是公益**，是**商業策略**（跟 Gmail 免費給 15GB 信箱、GitHub 開源專案不收錢、Canva 免費版的邏輯一樣）：
- 養大開發者跟小組織的使用習慣，等需求變大就會升級付費
- 服務一個小流量網站，他們的邊際成本接近零（伺服器本來就在跑）
- 越多網站在他們網路上 = 收集到的攻擊樣本越多 = 對企業客戶的安全服務越強

對我們這種小型 NGO 網站，免費版**綽綽有餘**。具體限制與我們實際用量：

| 項目 | 免費版上限 | 我們估計用量 |
|---|---|---|
| 每日網站請求次數 | 10 萬次 | < 500 次／日（剩 99.5% 餘裕） |
| 全球 CDN、HTTPS、DDoS 基礎防護 | 全部送 | ✅ 都用得到 |
| 流量／頻寬 | 小型網站無上限 | 無壓力 |
| 客服 | 只有社群論壇 | 我這邊維護即可 |
| 平台關停／改規則的風險 | 理論上有 | Workers 免費版上線 7 年沒變過規則，可信賴 |

**真正的「代價」是這兩件事**，協會需要評估：

1. **網址不夠正式** — `tiya-website.leepolung1.workers.dev` 結尾的 `.workers.dev` 一看就是「免費開發者平台」；加上 `leepolung1` 是我個人 Cloudflare 帳號的名稱，對外確實不夠協會感。
2. **帳號掛在一個人名下** — 我哪天離開、忘密碼、帳號被停用，網站會進入無人管狀態。這就是 E. 講的「誰持有」問題，只是場景換成 Cloudflare 帳號。

**過渡方案的具體做法（我可以馬上做）**：

1. 用協會官方 Gmail（`2024tiya@gmail.com`）開一個新的 Cloudflare 免費帳號
2. 新帳號註冊一個叫 `tiya`（或類似）的帳號子網域
3. 把網站從我的個人帳號搬到協會帳號，新網址變成像 `home.tiya.workers.dev`，「leepolung1」不見了
4. 協會官方 Gmail 的密碼本來就有多人共管，「誰持有」問題天然解決
5. 等之後有預算或補助下來，把 `tiya.org.tw` 接到這個帳號的上層 — **資料、設定不用搬，只是表面換招牌**

**取捨總結**：

| 項目 | 過渡方案 | 正式 `.org.tw` |
|---|---|---|
| 費用 | 0 元 | NT$800-1200／年 |
| 上線時間 | 我半小時內可弄好 | 視協會決議與註冊流程 |
| 網址質感 | `home.tiya.workers.dev` — 有 `.workers.dev` 尾巴，土砲味 | `tiya.org.tw` — 正式 NGO 形象 |
| Google 搜尋友善度 | 中等（`.workers.dev` 不是搜尋引擎最愛的網域類型） | 高 |
| 升級彈性 | 任何時候可加上正式網域，不用搬 | — |
| 對外發名片／媒體採訪信任感 | 過得去但不亮眼 | 加分 |

→ 例會結論：**先用過渡方案上線、有補助／預算下來再升級**，避免「沒錢就什麼都不動」的死結。

#### A. 要選哪一種網域？

| 網域類型 | 適合誰 | 信任感 | 限制 |
|---|---|---|---|
| **`.org.tw`** | **立案的台灣 NGO**（我們符合） | 高（看到就知道是台灣公益組織） | 需要協會「法人證號」才能申請 |
| `.com.tw` / `.tw` | 一般組織、公司 | 中（看不出是 NGO） | 無 |
| `.org` | 國際 NGO | 中（看不出是台灣） | 無 |
| `.com` | 通用 | 普通 | 無，但好的名字大多被搶光 |

**建議選 `.org.tw`** — 我們是台灣立案的協會，這是最對的網域類別。看到 `.org.tw` 大家就知道這是合法、台灣、非營利。

> 補充：協會「立案」這件事就是領了主管機關核發的「法人證號」（社團法人立案證書上會有編號）。申請 `.org.tw` 時要在表單裡填這個號碼，會自動線上驗證。

#### B. 去哪租？

`.org.tw` 由 TWNIC（財團法人台灣網路資訊中心）管理，但實際申請要透過「**註冊商**」（registrar）。常見的有：

| 註冊商 | `.org.tw` 年費 | 備註 |
|---|---|---|
| **戰國策** ([nss.com.tw](https://www.nss.com.tw/)) | NT$550 上下（最便宜的之一） | 介面中文、客服好找；多數小型 NGO 用 |
| **匯智 CloudMax** ([cloudmax.com.tw](https://domain.cloudmax.com.tw/)) | NT$760 上下 | 老牌、穩定 |
| **網路中文** ([netc.tw](https://netc.tw/)) | NT$800 上下 | 老牌 |
| **HiNet 中華電信** | NT$800 上下 | 最保守的選擇，缺點是介面老舊 |
| **GoDaddy 台灣** | NT$1000 上下 | 國際大廠，但 `.org.tw` 介面不是特別好 |
| **Gandi** ([gandi.net](https://www.gandi.net/)) | NT$700 上下（首年） | 國際註冊商，全英文介面 |

**建議**：選 **戰國策** 或 **匯智**，台灣本地客服、`.org.tw` 流程順、價格合理。
**不建議**：Cloudflare Registrar（雖然我們網站架在 Cloudflare 上，但他們的註冊商服務目前不支援 `.org.tw`）。

#### C. 一年大概多少錢？

- **網域年費**：NT$550-1000 / 年（看註冊商）
- **多年註冊有折扣**：建議一次買 2-3 年（NT$1100-3000），可以省事、避免漏約
- **隱私保護（WHOIS Privacy）**：有的註冊商免費送，有的要加 NT$200-500/年（避免註冊聯絡人的個資公開在 WHOIS 查詢）
- **總預算估計**：第一年含隱私保護大約 **NT$800-1200**

註：這只算網域本身。網站主機我們繼續用 Cloudflare 是免費的，所以 **官網的年度固定支出大約就是這個網域費**。

#### D. 申請要準備什麼？

1. **協會立案證書影本** — 我們已經有（社團法人立案時主管機關核發）。
2. **法人證號** — 立案證書上會寫，註冊時填進「統一編號／法人證號」欄位。
3. **聯絡資訊** — 協會的對外信箱（建議用協會的官方信箱 `2024tiya@gmail.com`，不要用個人信箱，避免成員離開後失聯）。
4. **付款方式** — 信用卡通常可以，協會公帳記得保留發票/收據。

申請流程：
1. 在註冊商的網站搜尋 `tiya.org.tw` 是否可用（沒被人註冊掉的話）→ 加入購物車 → 結帳。
2. 填 WHOIS 資料（含法人證號）。
3. TWNIC 後台自動驗證法人證號 → 通常幾分鐘到幾小時就核准。
4. 等註冊商通知「網域已開通」（通常一個工作天內）。

#### E. 用協會的帳號註冊就好

設定原則跟協會現在管理 Gmail 帳號的方式一樣 — 用 `2024tiya@gmail.com` 註冊、WHOIS 填協會名義（協會中文名、法人證號）、自動續約綁協會帳戶（不是個人卡）、把登入資訊存在協會的共享資料夾。

協會本來就有幾位核心成員持有官方 Gmail 的密碼，這套機制延伸到網域註冊商的帳號即可，**不需要為了網域多開一個信箱、也不必另外指派一個人專門掌管**。

#### F. 是否同時保留相近網域防搶註？

理論上應該。實務上 `.org.tw` 因為要法人證號才能申請，相對沒人會搶我們的名字。但如果預算允許，可以同時保留：

- `tiya.org.tw` — 主網域
- 不太需要：`tiya.tw`（任何人可註冊，但因為我們會主推 `.org.tw`，他人搶了傷害也不大）

**建議**：只買 `.org.tw` 就好，省錢。

#### G. 順便開「協會自己的信箱」 — 值得做的延伸

`@tiya.org.tw` 結尾的信箱（例如 `contact@tiya.org.tw`、`info@tiya.org.tw`）不是網域本身附贈的，要另外設定，但**這是延伸出來最有價值的一步** — 對外發信時的專業度直接升一檔，記者、合作對象收到 `@tiya.org.tw` 結尾的信會比 `@gmail.com` 更認真看待。

| 方案 | 費用 | 適合 |
|---|---|---|
| **Google Workspace for Nonprofits** | **免費**（NGO 限定，要申請） | 推薦 — 等同 G Suite 完整功能、用法跟 Gmail 一樣 |
| **Cloudflare Email Routing** | 免費 | 只能轉信，不能在 Gmail 介面用 `@tiya.org.tw` 寄信 |
| **戰國策／匯智自帶的 Email** | 含在網域月費裡 | 介面陽春，不建議 |

**建議**：先把網域租下來、確定可用，第二步就去申請 Google Workspace for Nonprofits（台灣立案 NGO 真免費，要填表審核大約 1-2 週）。獲准後協會就能擁有 `@tiya.org.tw` 信箱，現有 `2024tiya@gmail.com` 可以保留當備援。

#### H. 註冊好後的技術設定

註冊完成後，DNS／Cloudflare／網站接設定都是純技術工作，協會不需要管細節，由負責網站維護的成員執行即可（4 個步驟、總時間不到 30 分鐘）：
1. 在註冊商後台把 DNS 改為 Cloudflare 的 nameserver。
2. 在 Cloudflare 後台加入這個網域（Custom Domain 設定）。
3. 把 `tiya-website.leepolung1.workers.dev` 設定為 301 轉址到 `tiya.org.tw`（這樣舊網址連進來也會自動跳到新網址，SEO 不會掉）。
4. 更新網站程式碼裡 `astro.config.mjs` 的 `site:` 欄位、`public/robots.txt` 的 sitemap URL、README 的網址。

**從付完錢到網站能用新網址**：通常 24 小時內（DNS 全球生效需要時間）。

#### I. 給例會的具體決議事項

1. **要不要租？** → 建議：要。
2. **要選哪個名字？** → 推薦：`tiya.org.tw`（如果可用）。例會可以先確認其他候選名字。
3. **誰負責執行？** → 分兩塊：(a) 有協會證件的成員上註冊商網站完成購買；(b) 網站維護者接後續 DNS 與網站設定。
4. **預算？** → 第一年 NT$800-1200，由協會公帳支出。
5. **要不要也順便去申請 `@tiya.org.tw` 信箱？** → 建議：要，下次例會排程。

### 1.3 法律邊界（重要前提）

協會**尚未取得勸募字號**，所以網站上：
- ✅ 可以做：被動提供匯款資訊、收據說明、匯款後通知表單。
- ❌ 不能做：「Donate Now」「支持我們」這類號召性大按鈕、捐款金額示意（「100 元＝場地費」這類）、任何「請捐款」訊息。

這條約束會影響本報告中關於募款功能的時程規劃。

## 二、為什麼擁有屬於自己的官網很重要

社群帳號（FB、IG、Threads）已經能把訊息發出去，為什麼還需要花力氣維護一個官網？以下是**官網能做、社群做不到**的事：

### 2.1 訊息主導權

社群平台的演算法不是協會控制的。FB 一次調整，觸及率可以掉到原本的 10%。官網的版面、字句、出現順序、保留多久 — 全由協會自己決定。**Meta 哪天封了我們的帳號、IG 哪天倒了，故事還在自己的網域上。**

### 2.2 對外的「正門」

當記者要寫一篇關於新二代的報導、當立委助理要找移民議題的引述、當學者要做學術引用、當企業要找 CSR 合作對象 — 他們第一件事是 Google「台灣移民青年倡議陣線」。
**他們點進去看到的是 FB 還是官網，給的信任感差很多。** 官網 = 協會「正式存在」的招牌。

### 2.3 SEO（搜尋引擎最佳化）

社群貼文在 Google 搜不太到（FB 不開放給 Google 索引）。官網上的每一篇文章、每一次活動，都是長期被搜尋到的入口。
舉例：未來有民眾搜「新二代 身分認同」「移民工 政策」「東南亞 第二代 台灣」 — 官網有相關內容就會出現。

> 補充：這不是說社群就沒用了。**限時動態、即時活動快訊、互動式留言、跟群眾即時對話 — 社群仍然是主場**。重點是「不同類型的內容用不同管道」：永久性的東西放官網，即時／互動性的東西放社群。兩者互補，不是替代。

### 2.4 國際對話的入口

英文版讓我們有資格出現在國際移民倡議、人權報告、跨國 NGO 合作場合。
這在 IG／FB 上很難做到 — 對國際讀者來說，他們不熟悉中文社群平台，但他們會看英文網頁。

### 2.5 內容是資產，不是即時動態

社群貼文的生命週期是「幾天」。網站上的成員故事、活動紀錄、年度報告是「永遠」。
五年後新進的工作者、新的合作夥伴、新的研究者，都還能完整看到協會的歷程。

### 2.6 🗣 待討論：是否要做「身份分流」入口？

不同人來找協會的目的不同：個人想了解、媒體要採訪、學者要訪談、企業要合作、移民家庭想求助。
有些 NGO（例如 rethinktw）會在網站上做「**我是 ___，請點這裡**」式的入口分流。

這個做法的好處：訪客一進來就知道往哪走、節省雙方時間。
壞處：要協會內部有清楚的分工（誰負責回覆媒體、誰負責民眾求助、誰負責學術合作），不然分流後沒人接、反而更糟。

→ 例會可以決定：協會內部是否已經有人力／流程支援這種分流？如果還沒，這個功能就先不做。

## 三、改進建議 — 針對現有區塊

### 3.1 Hero 首屏

**目前狀況**：簡介段落以「我們是誰」描述為主。
**建議**：加一句**情感入口**式的標語。
可能方向（拋磚引玉，協會內部討論定調）：
- 「讓每一個新二代的聲音都被聽見」
- 「家鄉是兩個地方，認同是一個自己」

### 3.2 About 關於我們

目前已經有來源國家、核心價值。**現階段沒有特別要改動。**

### 3.3 News 媒體報導

**目前狀況**：2 篇卡片，左右滑動，圖片為主。
**建議**：每多一篇報導就累積一張卡，視覺上會更有「常被報導」的份量。**自然累積即可，不必勉強衝量。**

**🗣 待討論：要不要做「媒體採訪聯絡單」？**

這是什麼：一份 PDF（或網頁上一個小區塊），通常包含：
- 協會簡介一段（300 字以內）
- 對外發言窗口的聯絡方式（誰、什麼信箱、什麼時段可聯絡）
- 我們**可受訪的議題清單**（例如：新二代身分認同、移民工政策、勸募條例對小型 NGO 的影響…等，協會說了算）
- 高解析度的協會 logo 檔下載
- 過往媒體報導列表

為什麼有些 NGO 會做：當記者找上門時，**不用每次都從零開始解釋協會是誰、能受訪的人是誰**。記者拿到一份完整資訊，會更願意寫、寫得更準確。同時也讓協會內部有個對外口徑的統一版本。

要不要做：完全看協會內部**有沒有人力定期更新**（議題清單會變、可受訪人會變）。沒有的話，做了反而會變成過期資訊更傷信譽。

→ 例會討論：協會目前對外接受採訪的窗口分工是怎樣？如果還在摸索，這個就先不做。

### 3.4 Events 實體活動

**目前狀況**：嵌入 Facebook 貼文，每場活動一張卡，**現階段已經夠用。**
**可選的增強**（不急）：
- **觀眾回饋摘要**：每場活動結束後，把現場聽到的 1-2 句精彩回饋寫進卡片裡，比 FB 嵌入有溫度。
- **影音**：如果有錄活動 highlight，嵌入 YouTube／Vimeo 影片連結。

這些等有素材再加，不用為了加而加。

### 3.5 Column 天下獨立評論專欄

**目前狀況**：3 篇文章＋縮圖＋「我要投稿」CTA。
**建議**：
- 「我要投稿」目前的描述如果寫成「鼓勵成員投稿」，**改成「歡迎任何人投稿」** — 因為這個系列本來就不只開放給協會成員。
- 投稿區塊可以加一句「**我們提供寫作協助，沒寫過文章的也可以投**」 — 降低門檻，讓沒有書寫經驗、但有故事想說的人也敢投。

### 3.6 Donate 捐款資訊

**目前狀況**：法律免責 + 匯款資訊（待填）。協會帳戶已開戶。
**建議**：等帳戶資訊到位就把「待確認」換成實際資訊即可。**沒有急迫性，可以慢慢來。**

### 3.7 Social 社群連結

**目前狀況**：Facebook、Instagram、**Threads**（2026-05-17 加入）、Email。
**建議**：
- **YouTube／Podcast**：如果未來協會有產出影音內容，再加。

## 四、新功能建議 — 可以新增什麼

依**時程**分三組。每個建議都附「為什麼」「需要什麼」「會花多少功夫」。

### 4.1 短期（1-2 個月內、不需協會額外決策可以動）

#### A. 成員故事頁（「故事牆」）

每位（願意公開的）成員一張卡片：照片、原生國家、一段自述、一句最想對台灣社會說的話。

**為什麼**：協會名字是「**新二代留聲機**」 — 留聲機 = 記錄聲音的機器。網站上應該有最直接的體現。
**需要**：每位成員自己決定要不要公開、提供照片與一段文字。
**功夫**：技術上 1-2 天，資料蒐集視成員意願速度而定。

#### B. FAQ 常見問題頁

回答這些問題：「新二代的定義？」「TIYA 是什麼樣的組織？」「如何加入？」「如何投稿？」「為什麼還不能線上捐款？」

**為什麼**：減輕內部回覆訊息的工作量；同時為 SEO 加分（Google 喜歡 FAQ 結構）。
**需要**：協會校稿補充內容（我可以先寫一版草稿）。
**功夫**：技術上半天，內容定稿視協會回饋。

#### C. 站長後台流量觀察

讓協會內部能看到網站每日訪客數、訪客來源、哪些頁面最熱門、是用手機還是電腦看。

**為什麼**：對協會來說，這是「我們做的東西到底有沒有人看」的最直接答案。沒有數據就只能憑感覺。
**用什麼**：建議用 **Cloudflare Web Analytics**（我們本來就在 Cloudflare 上、免費、不需放 cookie、所以也不需要使用者同意彈窗、隱私友善）。
**功夫**：半天搞定。資料只有協會內部看得到，網站訪客不會察覺。

### 4.2 中期（取得勸募字號後）

#### A. 完整捐款流程

- 線上金流（信用卡 / Apple Pay / Line Pay）。建議走 **NetiCRM**（rethinktw 也用這個，NGO 友善的開源捐款系統）或 **奉獻箱 OurStable** 等台灣 NGO 平台。
- 捐款層級示意（取得勸募字號**之後**才能寫，例如：
  - 「500 元支持一場對話聚會的茶水費」
  - 「3000 元贊助協會半個月的辦公室／場地租金」
  - 「N 元贊助一場活動的講師車馬費／文宣印刷費」

  例子要等協會實際開支結構出來再校準。）
- 月定額會員方案。
- 收據自動寄送。

#### B. 年度成果報告（網頁版）

如果未來協會有意做年度報告，可以做成**互動式網頁**（含數據圖表、該年度精選故事、重要對外發聲時刻、來年方向預告）而不是只給 PDF 下載。

是否做、何時做都看協會年度規劃，**不是非做不可。**

### 4.3 長期願景（半年以上 / 看資源狀況）

> 這一節列的都是「**有資源才能做、現在不必煩惱**」的方向，當作可能性，不是承諾。

#### A. 東南亞語系擴充

協會服務的群體本來就涵蓋越南、印尼、泰國、菲律賓等新住民家庭。網站若能有越南語、印尼語等版本，至少首頁＋關於＋聯絡，象徵意義很大。

**門檻很高**：翻譯需要找母語者長期協助，而且每次內容更新就要多語同步更新，是長期的維護負擔。**非必要**。但如果未來有人力，是值得的「**有夢很美**」方向。

#### B. Podcast / 影音故事典藏

協會名稱「留聲機」本來就是聲音的隱喻。長期可以做：
- 不定期的成員口述故事 Podcast
- 對話活動的現場錄音精選

**門檻**：需要設備、剪輯時間、上架平台費用（Spotify／Apple Podcasts），對小型 NGO 是長期投入。**列入「如果未來有預算可以做」的方案。**

**國際參考**：[StoryCorps](https://storycorps.org/)（美國民間口述歷史 NGO，純粹靠錄音故事撐起一個組織）。

#### C. 議題搜尋連動的內容地圖

網站上的所有內容（文章、活動、故事）依議題（升學、勸募、認同、家庭、政策…）交叉索引。讀者點「政策」可以看到所有相關文章＋活動＋成員故事。

需要等內容累積到一定量（30+ 篇）才有意義。

## 五、參考案例

### 5.1 國內 — 婦女救援基金會（<https://www.twrf.org.tw/>）

**值得學習**：
- 議題（人口販運、性暴力、慰安婦、新移民）每個都有獨立的議題頁＋資源庫。
- 故事為核心：當事人匿名敘事、影像紀錄。
- 線上捐款流程做得完整（取得勸募字號後可以參考）。

### 5.2 國際 — Define American（<https://defineamerican.com/>）

**為什麼參考**：美國移民倡議 NGO，由移民出身的記者 Jose Antonio Vargas 創辦。**和 TIYA 任務最像。**
**值得學習**：
- 用個人故事推動議題的方式。
- 與媒體（電視劇、新聞媒體）的合作模式：他們有專屬的「Hollywood 合作」頁面，幫助影視作品準確呈現移民議題。

### 5.3 國際 — StoryCorps（<https://storycorps.org/>）

**為什麼參考**：純口述歷史 NGO，名字也跟「留聲機」精神共鳴。
**值得學習**：每段錄音都有對應的網頁卡片＋文字摘要＋背景介紹。是「**聲音故事典藏**」的標竿。

## 六、給例會的討論題（依優先程度分三層）

依**重要程度**重新排序，避免大家被一長串題目淹沒。

### 🔴 重要 — 例會建議當天討論並做方向決議

**1. 網域** ｜ 見 1.2
要不要正式租用 `tiya.org.tw`？預算 NT$800-1200 / 年，誰來執行登記？是否同時申請 `@tiya.org.tw` 信箱？
*為什麼重要*：這是「官網正式存在」的最後一步，越早決定越好（避免名字被人搶註冊、避免今天決定明天又拖半年）。具體選項、流程、誰來做我都在 1.2 寫清楚了，例會可以照著走。

**2. Hero 首屏標語** ｜ 見 3.1
要不要正式定一句對外口號？（現在是描述句）
*為什麼重要*：這是訪客打開官網第一眼看到的話，等於協會對外的「核心訊息」。需要協會內部對齊，不能由我這個技術人員自己決定。

### 🟡 許願 — 給成員許願，看意願再做

**3. 成員故事頁（「故事牆」）** ｜ 見 4.1 A
讓每位成員一張卡：照片、原生國家、自述、想對台灣社會說的話。
*為什麼是許願*：這完全看成員自己**願不願意公開**，不能勉強。例會可以開放給大家舉手「我願意」「我先觀察」，有多少做多少，沒有壓力。

### ⚪ 非必要 — 想聊就聊，不聊也沒差

**4. 身份分流入口** ｜ 見 2.6
「我是個人／媒體／學者」分流頁面。
*為什麼非必要*：需要協會內部有清楚的窗口分工（誰回媒體、誰接民眾求助、誰處理學術合作）才有意義。如果分工還在摸索，這個就不要急。

**5. 媒體採訪聯絡單** ｜ 見 3.3
PDF 形式的協會簡介＋對外發言窗口＋可受訪議題清單，給記者參考。
*為什麼非必要*：需要協會有人**定期更新**才有用（議題清單會變、可受訪人會變）。沒人力維護的話做了反而過期傷信譽。
