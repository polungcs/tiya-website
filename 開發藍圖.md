# Part 1  ·  為什麼要架設網站

## 1.1 我們組織現在的狀況

我們已經正式立案為「台灣移民青年倡議陣線協會」，目前主要的對外曝光集中在三個地方：

- Facebook 粉絲專頁
- Instagram 帳號
- 天下獨立評論專欄（投稿平台，不是我們自己的）

這三個管道各自有它的優點，但合在一起仍然有幾個結構性的限制——這些限制不是「再多發幾篇貼文」就能解決的。

## 1.2 社群媒體做不到的七件事

這部分要回答的問題是：「FB 和 IG 已經有了，為什麼還要多花力氣做網站？」

### ① 不被演算法綁架

在 FB／IG 上，貼文要被誰看到、看到多少，是由 Meta 的演算法決定。我們發了一篇活動公告，可能 100 個追蹤者裡只有 10 個看得到。

網站不一樣——只要對方知道網址、或在 Google 搜得到，他就一定看得到。我們的訊息被誰看到，由我們決定。

### ② 可以被 Google 搜尋到

當記者、學者、學生、或潛在合作對象 Google「移民二代 倡議 台灣」「新二代留聲機」這類關鍵字時，社群貼文很難被搜到，但網站會。這對 NGO 接觸到「不認識我們但需要我們」的人，是非常關鍵的。

### ③ 內容會永久保存且可以被結構化

FB 貼文滑下去就找不回來。同一個人三個月後想再讀一次某篇文章，幾乎不可能。

網站可以把所有活動依照時間排序、把專欄文章分主題分類、把組織的歷年大事記做成時間軸——這些「結構化的呈現」是社群媒體做不到的。

### ④ 雙語（甚至多語）支援

我們的核心關懷之一是跨語言群體。在 FB／IG 上要做雙語並陳很困難（每篇貼文要重複多次、版型亂掉、追蹤者體驗破碎）。

網站可以一鍵切換語言，讓英文讀者讀到自己語言的內容。這對倡議組織來說是「核心」而不是「加值」。

### ⑤ 對外的正式形象與信任感

「你們有官網嗎？」這句話，是記者、學界、政府、補助單位、潛在合作組織最常問的第一句話。

一個正式立案的協會如果連網站都沒有，在許多正式場合會被默認為「比較不正式」。一個結構清楚、設計專業的網站本身就是一張名片，可以大幅提升對外接洽的效率與可信度。

### ⑥ 不依賴單一平台

FB 帳號可能被檢舉停權、IG 演算法可能突然改、平台政策可能改變。如果我們所有的對外資料都在 Meta 上，等於把組織的對外存在權全部交給一家美國公司。

有自己的網站，等於有一個「無論其他平台發生什麼事，我們都還在」的核心據點。這對倡議組織尤其重要。

### ⑦ 募款、報名、徵文、聯絡的整合入口

未來我們可能會有：捐款、實體活動報名、徵文投稿、加入志工、申請合作、媒體聯繫……這些動作如果分散在 FB 私訊、IG DM、Email 之間，會非常混亂；網站可以把這些整合成一個「進來什麼都找得到」的入口。

## 1.3 對照表：FB／IG vs. 網站

| 能力 | FB | IG | 網站 |
|---|---|---|---|
| 能被 Google 搜到 | 弱 | 幾乎不行 | 強 |
| 所有人都看得到（不被演算法限流） | ✗ | ✗ | ✓ |
| 訊息能依主題分類、長期保存 | ✗ | ✗ | ✓ |
| 雙語／多語切換 | ✗ | ✗ | ✓ |
| 對外提供正式組織形象 | 中 | 中 | 強 |
| 可整合捐款、報名、徵文等動作 | 弱 | 弱 | ✓ |
| 不依賴第三方平台 | ✗（依賴 Meta） | ✗（依賴 Meta） | ✓ |
| 即時、容易擴散、有社群互動 | ✓ | ✓ | 弱 |
| 影音、限動、互動貼文 | 中 | ✓ | 弱 |

| 結論 網站不是要取代 FB／IG，而是補上它們做不到的東西。 FB／IG 負責「即時擴散與互動」，網站負責「長期保存、正式呈現、被搜尋到、整合行動入口」。三者要分工、不是替代。 |
|---|

# Part 2  ·  網站會長什麼樣子

## 2.1 整體結構

第一階段預計上線六個主要頁面：

| 頁面 | 功能 | 對誰開放更新權限 |
|---|---|---|
| 首頁 | 組織形象、最新動態、導向其他頁面的入口 | 管理員 |
| 關於我們 | 組織介紹、立案資訊、成員組成、宗旨 | 管理員 |
| 實體活動公告 | 歷年活動、未來活動、活動回顧、照片 | 所有核心成員 |
| 天下獨立評論專欄 | 所有專欄文章列表＋外部連結 | 所有核心成員 |
| 支持我們（捐款） | 捐款說明、匯款資訊、未來金流串接 | 管理員 |
| 社群連結 | FB／IG／Email 整合入口 | 管理員 |

## 2.2 每個頁面的細節

### 頁面一：首頁（Home）

這是訪客看到的第一個畫面，要在 5 秒內回答：「這是什麼組織？我為什麼要看下去？」

包含的區塊：

- 頂端：語言切換條（繁中／英文）
- 導覽列：含正式 Logo 和主要頁面連結
- Hero 區塊：組織名（中／英）、一句話介紹
- 關於我們摘要：組織簡介＋來源國家標籤
- 最新／近期實體活動卡片
- 天下獨立評論專欄文章列表
- 支持我們入口（連到捐款說明頁）
- 社群連結（FB／IG）
- 頁尾：聯絡資訊、版權
### 頁面二：關於我們（About）

回答「你們是誰」這個最基本的問題，是給記者、合作對象看的標準頁面。

包含的區塊：

- 組織宗旨與定位
- 協會立案資訊（協會登記名稱、立案年份）
- 成員組成的多元性（來自越南、中國、泰國、印尼、菲律賓、馬來西亞、柬埔寨、寮國、緬甸）
- （未來可加）核心成員介紹（同意公開的成員）
- （未來可加）組織大事記時間軸
### 頁面三：實體活動公告（Events）

這是更新最頻繁的頁面，會持續累積我們的活動紀錄。

每場活動會記錄：

- 活動名稱、日期、地點
- 講者介紹
- 活動主題與簡介
- 活動照片（合照、現場照）
- hashtag、合作場地
- （如果有）報名連結（目前用 Google 表單）

| 為什麼這個頁面特別重要 我們組織的活動現場其實是組織能量最強的地方，但 FB／IG 的限制讓這些活動發生過後就「沉到動態下面」。網站的活動頁可以讓「2025 年 9 月那場宜蘭原新混血討論會」十年後都還能被找到——這是組織的歷史檔案。 |
|---|

### 頁面四：天下獨立評論專欄（Column）

我們組織成員在天下獨立評論有專欄，這個頁面把所有文章彙整起來，每篇文章顯示標題、作者、發表日期，點擊後連到天下原始文章。

這樣做的好處是：訪客在我們網站上看到「我們組織的成員寫了哪些文章、各自寫什麼主題」的全貌，而不需要去天下站內慢慢翻。

### 頁面五：支持我們 / 小額募款（Donate）

對 NGO 而言，「能不能收到捐款」是組織能否永續的關鍵；網站要做為「邀請支持的入口」。

為什麼 NGO 一定要有捐款頁？

- 捐款本身就是一種對組織理念的「具體支持」，比按讚分享更有意義
- 有些人是先讀到文章、被打動後才想捐款——這個瞬間如果沒有捐款入口，那筆捐款就消失了
- 申請補助、合作時，「我們有獨立募款管道與紀錄」是非常加分的條件
- 捐款者本身會變成組織的長期關係人——他們未來可能也是志工、媒體、合作橋樑
這個頁面要呈現的內容：

- 為什麼要捐款給我們：簡短說明組織宗旨、捐款會支持哪些事（活動辦理、營運、未來發展）
- 捐款用途透明說明：例如「100 元可以支持……、500 元可以支持……、1000 元可以支持……」
- 捐款方式：銀行匯款資訊（戶名、銀行、帳號）、郵局劃撥、（未來）線上刷卡與電子支付
- 收據說明：是否能開立收據、收據抬頭如何處理（這對個人申報所得稅扣除很重要）
- 捐款後請告知我們的方式：例如填寫一份 Google 表單告知匯款資訊以利對帳
- 感謝詞、捐款人致謝（若捐款人同意公開）

| 分階段實作策略 第一階段（網站上線當下）：放捐款說明 + 匯款資訊 + 通知 Google 表單。完全免費、零金流串接。 第二階段（網站運作 3－6 個月後）：依實際捐款量，評估是否要串接金流（綠界 ECPay、藍新 NewebPay 等台灣金流商）讓人可以線上刷卡。 第二階段才做的原因：金流串接需要協會出示立案文件、開立特約商店帳戶、簽約，並有手續費（每筆 2.7%~3% 起跳）。如果第一階段每月只收到幾筆小額捐款，根本不划算。先用「銀行匯款」累積數據，再決定要不要花錢串金流。 |
|---|

第二階段：金流串接的選項與費用

| 金流方案 | 申請門檻 | 手續費 | 適合 |
|---|---|---|---|
| 綠界 ECPay | 需協會立案文件、開戶 | 信用卡 2.75%、ATM 每筆 NT$10 | 中小型 NGO，台灣最常見 |
| 藍新 NewebPay | 需協會立案文件、開戶 | 信用卡 2.65~2.85% | 與綠界類似，差別不大 |
| LINE Pay 商家 | 需立案文件＋實名審核 | 信用卡 2.8% | 年輕族群、行動支付習慣者 |
| PayPal（國際捐款） | 免立案，但提現有限制 | 約 4.4% + 固定費用 | 如果有海外捐款需求 |
| 街口支付 | 需立案 | 約 2%~3% | 本土小額頻繁支付 |

提醒：金流商的合約都有「最低月費」或「最低保證手續費」條款，要先看清楚。如果月捐款額很低，可能繳出去的手續費比收到的捐款還多。

### 頁面六：社群連結（Connect）

整合 FB、IG、Email 的單一入口頁面，方便分享。

## 2.3 語言版本

考量翻譯工程量，第一階段語言策略簡化為——

| 第一階段語言計畫 1. 主語言：繁體中文（完整內容） 2. 次語言：英文（完整內容） 3. 其他東南亞語言：第一階段不主打、不列為里程碑。等繁中與英文兩版穩定後，視組織人力與翻譯志工資源，再分批加入越南文、印尼文、泰文等。  這樣做的好處：把有限的精力先集中在「能做好的兩種語言」上，而不是 11 種都做半套。 |
|---|

技術上，網站架構從一開始就會「準備好」支援多語擴充——也就是說，未來想加越南文、泰文時，只需要『翻譯內容』，不需要『改寫網站』。這是技術選型階段就要考慮的事。

# Part 3  ·  開發藍圖（技術細流）

## 3.1 重大技術抉擇：靜態 vs 動態網站

在進入細節前，要先決定一件影響後面一切的事：網站要做成「靜態」還是「動態」？這個選擇會影響費用、開發複雜度、未來能做什麼功能。

### 先解釋兩者的差別

靜態網站（Static Site）

所有頁面都是事先做好的 HTML 檔案。使用者瀏覽時，伺服器只是「把現成的檔案發給你」，不需要每次重新計算。

生活類比：像「印好的書」——所有內容都已經印在紙上，誰來翻都看到一樣的內容。要更新內容就要重新印一次。

動態網站（Dynamic Site）

頁面是使用者打開時，由伺服器即時生成的——伺服器會去資料庫裡撈資料、組合成 HTML、再發給使用者。

生活類比：像「現點現做的料理」——每個人來，廚房才開始煮。可以根據點餐者的口味做客製化。

### 兩種方案完整對照

| 比較項目 | 靜態網站（Astro + Decap CMS） | 動態網站（如 WordPress、Strapi、自製後端） |
|---|---|---|
| 首年費用 | NT$800（網域） | NT$3,000~6,000（網域＋伺服器） |
| 第二年起年費 | NT$800 | NT$3,000~5,000 |
| 伺服器需求 | 不需要（檔案放 Netlify 免費 CDN） | 必須租虛擬主機 VPS 或共享主機 |
| 速度 | 極快（CDN 全球分發） | 中等（需查詢資料庫） |
| 安全性 | 極高（沒有後端可被攻擊） | 中等（資料庫、後台都是攻擊面） |
| 維護負擔 | 低（每年只要續網域） | 高（要更新系統、修補漏洞、備份資料庫） |
| 流量爆增的承受力 | 強（CDN 自動擴展） | 弱（伺服器會被打掛，要升級方案） |
| 即時互動功能 | 有限 | 強 |
| 使用者帳號／登入 | 難（要靠第三方） | 容易 |
| 留言／論壇 | 難（要靠第三方） | 容易 |
| 金流串接 | 可以（透過第三方服務） | 可以（更彈性） |
| 搜尋功能（站內） | 需另外設定 | 內建容易做 |
| 編輯內容方便度 | 中等（透過 CMS） | 高（後台像 Word） |
| 技術門檻 | 中（要懂 Git、Markdown） | 低～高（看用什麼） |
| 搬遷／備份 | 極簡單（一份 Git repo 就是全部） | 複雜（要備份資料庫、設定檔） |

### 各自適合什麼樣的組織

靜態網站適合：

- 內容為主、互動為輔的網站（媒體、組織官網、作品集）
- 預算有限、人力有限的組織
- 不需要使用者登入、留言、即時互動的網站
- 希望「設好之後不太需要管」的組織
動態網站適合：

- 有大量動態內容、使用者互動需求的網站（社交平台、會員制服務）
- 需要使用者註冊、登入、個人化內容的網站
- 有預算長期養伺服器、有人力處理資安的組織
- 有獨特功能需求、第三方服務無法滿足的組織
### 這對我們組織意味著什麼？

| 務實分析 我們現階段需要的是：對外形象、活動公告、文章彙整、捐款入口、聯絡。這些都是「靜態網站」可以做得很好的事。 我們不需要的是：使用者登入、會員系統、即時聊天、論壇、複雜搜尋。 現實面：我們組織懂技術的人不多。動態網站要長期維護伺服器、處理資安、定期備份，這在「人力極度有限」的情況下風險很高。如果哪天柏龍比較忙、伺服器掛了沒人發現，網站就會出事。 所以建議：第一階段做靜態網站。如果未來真的需要動態功能（例如要做會員制、要做即時投票），可以再從靜態升級到混合模式（靜態為主＋小部分動態 API）——這在技術上是可以漸進式做的，不需要打掉重練。 |
|---|

### 動態網站方案的詳細費用（如成員會議決定走動態）

| 項目 | 費用 | 說明 |
|---|---|---|
| 網域（.org.tw） | NT$800／年 | 同靜態方案 |
| 共享主機（入門） | NT$1,200~3,000／年 | Bluehost、Hostinger、台灣本土如戰國策；資源有限，流量大就會卡 |
| VPS 虛擬主機（中階） | NT$3,000~7,200／年 | DigitalOcean、Linode、Vultr 等；自由度高，需自己管伺服器 |
| WordPress 託管服務 | NT$4,800~12,000／年 | Kinsta、SiteGround、Cloudways；幫你管 WordPress，省事但貴 |

## 3.3 字體選擇（獨立的選項）

字體不影響功能，純粹是視覺品質的選擇。可以獨立決定。

| 字體 | 費用 | 視覺感 | 建議使用情境 |
|---|---|---|---|
| 思源宋體 | 免費 | 中規中矩、現代、可讀 | 預算優先、所有正式場合都可 |
| JF 蘭陽明體 | NT$2,200~3,000 一次買斷 | 有人文氣息、優雅、文化感 | 想突出「文學感」「人文感」的組織 |
| Noto Sans TC（思源黑體） | 免費 | 現代、清晰 | 搭配宋體做標題／內文層次 |

柏龍的看法：思源宋體＋思源黑體已經完全夠用且免費。蘭陽明體是「願不願意為視覺氣質多花一筆」的選擇，組織以後做平面物料（海報、簡報）也會用到，平攤下來不貴。但這純粹是選項，不是必要。

## 3.4 名詞解釋（給非技術成員）

CDN（內容傳遞網路）

分散在世界各地的伺服器網絡。當有人從日本連我們網站時，會從最近的日本節點拿到資料；從加拿大連則從加拿大節點拿。這讓網站全球載入速度都很快。Netlify 內建免費 CDN。

CMS（內容管理系統）

讓非技術人員可以「像在 Word 編輯」一樣更新網站內容的工具。我們選 Decap CMS 是因為它免費、且和我們選的整個技術組合相容。成員不需要學程式，直接用網頁後台填表單就能新增活動、上傳照片。

版本控制（Git／GitHub）

簡單說就是「網站所有修改的歷史紀錄都會留下來」。誰在什麼時候改了什麼，都可以查、可以回復。任何人手滑刪錯東西，都救得回來。

部署（Deploy）

把網站從『開發者電腦』搬到『全世界都能連上的伺服器』的過程。我們用 Netlify，每次內容更新都會自動部署，不需要手動操作。

網域（Domain）

就是網址，例如 example.org.tw。這是唯一需要每年付費的項目。.org.tw 是台灣為立案非營利組織保留的網域後綴。

VPS（虛擬私人伺服器）

「動態網站」如果走自架路線，就需要 VPS。它是租來的一台虛擬電腦，24 小時開著替你執行網站。每月大約 NT$300~600 起跳。

## 3.5 開發階段

以下是建議方案（靜態網站）的開發流程，預計 1至 2個月內第一版上線。

### 階段 0：環境準備（第 1 週）

負責人：開發者

要做的事：

- 申請 GitHub 帳號（或用組織帳號）並建立組織用的 repository
- 申請 Netlify 帳號並連結 GitHub
- 在本機安裝 Node.js、Git、VS Code
- 建立 Astro 專案的初始骨架，把它推上 GitHub，看 Netlify 是否能自動部署成功
### 階段 1：版型設計與首頁開發（第 1 週）

負責人：開發者＋成員提供視覺意見

要做的事：

- 依照已經做好的 demo 樣板建立首頁
- 確認最終配色、字體、Logo 使用規範
- 實作雙語切換（繁中／英文）
- 建立 Header、Footer 等共用元件
### 階段 2：其他頁面開發（第 1週）

負責人：開發者

要做的事：

- 建立關於我們頁面
- 建立實體活動列表頁＋單一活動頁
- 建立天下獨立評論文章列表頁
- 建立支持我們（捐款）頁面（第一版：說明＋匯款資訊＋通知表單）
- 建立社群連結頁
- 確認所有頁面的 RWD（手機、平板、電腦顯示都正常）
### 階段 3：CMS 整合（第2週）

負責人：開發者

要做的事：

- 申請 .org.tw 網域並驗證組織立案文件
- 安裝設定 Decap CMS
- 為「實體活動」「文章列表」「組織資訊」分別建立 CMS 編輯介面
- 設定編輯權限
- 實際讓一位非技術成員試用 CMS 後台、回收修改建議
### 階段 4：英文版內容填入（第 3週）

負責人：開發者＋翻譯志工

要做的事：

- 先完成繁中版所有靜態內容（介面文字、固定頁面內容）
- 組織內部募集懂英文的成員協助翻譯與校對
- 完成英文版上線
### 階段 5：上線前測試與正式發布（第 4週）

要做的事：

- 跨瀏覽器測試（Chrome、Safari、Firefox、手機 Chrome）
- 無障礙檢測（螢幕閱讀器、鍵盤導覽、色彩對比）
- SEO 設定（網站描述、社群分享圖、sitemap、robots.txt）
- Google Search Console 註冊並送出 sitemap
- 正式對外發布——透過 FB、IG、組織 Email 公告
### 階段 6（未來）：金流串接（上線 3－6 個月後評估）

不在第一版範圍。等網站穩定運作、累積實際捐款數據後再評估。

### 階段 7（未來）：擴充其他語言版本

等繁中＋英文兩個版本穩定運作後，視組織需求與翻譯人力，逐一加入越南文、印尼文等。

## 3.8 上線後的維運

### 開發者的工作（柏龍）

- 處理偶發的技術問題（每月平均應該不超過 1－2 小時）
- 依組織需要新增頁面、調整版型
- 教育並支援其他成員使用 CMS
### 核心成員的工作

- 每場活動結束後，由負責人在 CMS 後台新增該場活動紀錄
- 每篇新發表的天下評論文章，加進文章列表
- 發現網站任何錯字、錯誤資訊、壞掉的連結，回報或自己直接在 CMS 改
- 每月對帳一次捐款進帳，更新到組織財務紀錄
### 組織管理階層（理事會）的工作

- 每年續訂 .org.tw 網域
- 把網站納入組織年度預算（NT$800／年的網域費）
- 確保「網站總管理員 Email」與密碼是組織擁有
- 每年檢視一次網站內容，確認立案資訊都是最新的
- 每年檢視一次捐款收支，決定要不要進入「第二階段金流串接」
## 3.9 給核心會議的具體決策清單

這份提案要請成員會議拍板的事項：

- 是否同意啟動網站建置案？

- 是否同意第一年預算（靜態最省 NT$800、靜態建議 NT$3,000~3,800、動態入門 NT$3,000~5,000）？
